🎯 Prácticas Detalladas

Práctica 1: Simulación CoppeliaSim

Objetivo: Integración básica del simulador CoppeliaSim con ROS 2. Adquisición de datos de odometría y LiDAR.

Aspecto Detalles
Nodos CoppeliaSimNode, LifecycleManager
Topics Publicados PUB: /odometry PUB: /scan
Topics Suscritos SUB: /cmd_vel SUB: /pose (opt)
Frecuencia 50 Hz (sensores)
Complejidad ⭐ Mínima (2 nodos)

Práctica 2: Localización con Filtro de Partículas

Objetivo: Implementar localización bayesiana que combine odometría y LiDAR para estimar pose del robot.

Aspecto Detalles
Nodos Nuevos ParticleFilterNode
Topics Publicados PUB: /pose
Topics Suscritos SUB: /odometry (sync) SUB: /scan (sync)
Algoritmo Filtro Bayesiano: Predicción → Actualización → Remuestreo → Estimación
Parámetros particles=1000, σ_v=0.05, σ_w=0.15, σ_z=0.10
Complejidad ⭐⭐ Media (3 nodos, sincronización)

Práctica 3: Planificación y Control

Objetivo: Sistema completo de navegación autónoma con planificación de caminos y control mediante Pure Pursuit o Wall Following.

Aspecto Detalles
Nodos Nuevos PRMNode, PurePursuitNode, WallFollowerNode
Topics Publicados PUB: /path PUB: /cmd_vel
Planificación Probabilistic Roadmap: 250 muestras, radio conexión 1.5m
Control Pure Pursuit (seguimiento) ó Wall Following (reactivo)
Frecuencias PRM: 10 Hz, Controlador: 50 Hz
Complejidad ⭐⭐⭐⭐ Alta (5 nodos, loop cerrado)

Práctica 4: Robot Real TurtleBot3

Objetivo: Implementación en hardware real con teleopeación manual y monitoreo de objetivos.

Aspecto Diferencia vs P3
Nodos Nuevos OdometryNode, KeyboardNode, TeleoperationNode, MonitoringNode
Fuentes de Datos TurtleBot3 /odom + LDS-01 real (vs CoppeliaSim)
Tipo cmd_vel geometry_msgs/Twist (vs TwistStamped)
Parámetros particles: 1000→2000, ruido 2x mayor
Nuevas Características Teleopeación (w/s/a/d), Detección de obstáculos, Monitoreo de meta
Complejidad ⭐⭐⭐⭐⭐ Muy Alta (9 nodos, hardware real)

Referencia de Estándares ROS 2

  • Tipos de Mensajes Estándar: std_msgs, geometry_msgs, nav_msgs, sensor_msgs
  • Lifecycle Pattern: unconfigured → inactive → active → finalized
  • Message Filters: Sincronización temporal de múltiples tópicos
  • Quality of Service (QoS): Configuración de confiabilidad y durabilidad
  • Parameter Server: Configuración dinámica de parámetros
